# 🌌 Weby HomeLab: The Infrastructure Matrix

Ласкаво просимо до центрального репозиторію моєї домашньої лабораторії та хмарної інфраструктури. Це не просто набір серверів — це автоматизована, безпечна та відмовостійка екосистема, яка поєднує публічні хмарні вузли з локальними кластерами Proxmox через зашифровані тунелі.

Тут зберігається код, конфігурації (Infrastructure as Code) та архітектурні рішення, які керують усім: від моніторингу якості повітря та електроенергії в Києві до захищеної VoIP-телефонії.

---

## 🏗 Архітектура Системи (Overview)

Інфраструктура розділена на кілька логічних зон, які безперебійно взаємодіють одна з одною через **Tailscale Mesh VPN** та **Cloudflare Tunnels**.

### ☁️ Cloud Edge (Hetzner & IONOS)
Публічний периметр, який приймає зовнішній трафік. Максимально захищений за допомогою `NFTables` (Drop Policy) та `Fail2Ban`.
*   **Сервіси:** Docker-хости, Asterisk VoIP (з жорсткими обмеженнями PJSIP), Turnserver, Uptime Kuma для зовнішнього моніторингу.
*   **Доступ:** Нестандартні SSH-порти (54322), вхід виключно за ED25519-ключами, форсовані `tmux`-сесії для надійності роботи.

### 🏠 Local Core (Proxmox VE Cluster)
Серце домашньої лабораторії. Віртуалізоване середовище, яке керує внутрішньою мережею та збереженням даних.
*   **Ноди:** Надійний сервер (pve02) та бекап-нода на базі лептопа (pve01) з автоматичним моніторингом батареї.
*   **Контейнери (LXC):** Мережева безпека (AdGuard Home), IT-Tools, NAS (SMB/NFS), ізольовані середовища для розробки.

## 🚀 Поточні Реалізації (Flagship Projects)

Ці проєкти вже розгорнуті, активно працюють та приносять щоденну користь.

### 🔆 Light Monitor Kyiv (v1.2.1)
*   **Статус:** 🟢 Активно | [Репозиторій](https://github.com/weby-homelab/light-monitor-kyiv)
*   **Суть:** Інтелектуальна система моніторингу електроенергії з Web-дашбордом (графіки Plan vs Fact) та Telegram-ботом.
*   **Фічі:** Аналіз графіків на 48 годин, виявлення відхилень від розкладу (точність увімкнень/вимкнень) та формування детальних добових і тижневих звітів.

### 🛡️ Security Monitor Kyiv (v1.2.1)
*   **Статус:** 🟢 Активно | [Репозиторій](https://github.com/weby-homelab/security-monitor-kyiv)
*   **Суть:** Внутрішній дашборд безпеки (доступний лише через тунель).
*   **Фічі:** Real-time показники якості повітря (AQI, PM2.5/10 через OpenMeteo та SaveEcoBot), інтеграція з Ubilling API, жива мапа тривог (alerts.in.ua).

### 📞 Secure VoIP Core
*   **Статус:** 🟢 Активно (Контейнеризація)
*   **Суть:** Розгортання захищених Asterisk-серверів на базі Docker. Автоматизоване встановлення, резервне копіювання та захист від брутфорсу на рівні ядра мережі (NFTables).

## 🗺️ Дорожня Карта та Плани (Roadmap)

Проєкт постійно еволюціонує. Структура цього репозиторію створена як фундамент для майбутніх покращень:

### Фаза 1: Infrastructure as Code (IaC) 🛠️
*   [ ] **Ansible (`configs/`):** Перехід від bash-скриптів до плейбуків для автоматичного розгортання правил NFTables, налаштувань SSH та Docker-середовищ.
*   [ ] **Terraform:** Декларативне створення VPS на Hetzner та IONOS.

### Фаза 2: Глибока Обсерваваність (Observability) 📊
*   [ ] Розгортання стеку **Grafana + Prometheus + Loki**.
*   [ ] Збір телеметрії з Proxmox (температури, знос дисків) та хмарних серверів (навантаження на SWAP, RAM) в єдиний дашборд.

### Фаза 3: CI/CD пайплайни (`pipelines/`) ⚙️
*   [ ] Налаштування GitHub Actions для автоматичного тестування коду перед деплоєм (наприклад, автоматичний лінтинг Python-коду в `light-monitor-kyiv`).
*   [ ] Автоматична збірка та пуш Docker-образів для власних сервісів.

### Фаза 4: High Availability (HA) & Backups 💾
*   [ ] Повне автоматизоване бекапування конфігурацій LXC-контейнерів у хмару.
*   [ ] Тестування сценаріїв Disaster Recovery (відновлення інфраструктури "з нуля" за допомогою збережених конфігів).

## 📂 Структура Репозиторію

*   📂 `docs/` — Архітектурні рішення (ADR), інструкції та документація.
*   📂 `configs/` — Конфігураційні файли (Ansible, Docker Compose, конфіги сервісів).
*   📂 `scripts/` — Утиліти для автоматизації (bash/python скрипти для аудитів безпеки, моніторингу SWAP тощо).
*   📂 `pipelines/` — Файли конфігурації CI/CD (GitHub Actions).
*   📂 `diagrams/` — Топології мереж та архітектурні схеми.
*   📂 `tests/` — Скрипти перевірки безпеки та цілісності інфраструктури.
